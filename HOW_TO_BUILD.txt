1. Собери игру (self-contained, чтобы у игрока не требовался .NET):
dotnet publish -c Release -r win-x64 --self-contained true

Папка publish (bin\Release\net10.0-windows\win-x64\publish\) — там Wrecksmith.exe + все .dll + рантайм. Это и есть твой билд. Не single-file — обычная папка заливается чище.

2. Скачай Steamworks SDK (partner.steamgames.com → Documentation → SDK), распакуй. Иди в sdk\tools\ContentBuilder\. Там подпапки builder\ (steamcmd), content\, scripts\, output\.

3. Положи содержимое publish-папки в content\ — то есть content\Wrecksmith.exe, content\*.dll и т.д. content\ = корень того, что получит игрок.

4. Узнай реальный DepotID: App Admin → SteamPipe → Depots. Не угадывай, возьми точное число оттуда.

Wrecksmith (4861360)

5. Создай два скрипта в scripts\.
app_build_4861360.vdf:
"appbuild"
{
    "appid"   "4861360"
    "desc"    "Wrecksmith alpha"          // внутреннее описание, игроки не видят
    "buildoutput" "..\output\"
    "contentroot" "..\content\"
    "setlive" ""                          // пусто = не делать live сразу (безопаснее)

    "depots"
    {
        "4861361" "depot_build_4861361.vdf"
    }
}


depot_build_ТВОЙ_DEPOTID.vdf:

"DepotBuild"
{
	// Set your assigned depot ID here
	"DepotID" "4861361"

	"FileMapping"
    {
        "LocalPath" "*"
        "DepotPath" "."
        "recursive" "1"
    }
    "FileExclusion" "*.pdb"
}


Замени ТВОЙ_DEPOTID на число из шага 4 (и в имени файла тоже).
6. Залей. Из папки builder\:
steamcmd.exe +login mm_makarov +run_app_build "..\scripts\app_build_4861360.vdf" +quit
steamcmd.exe +login mm_makarov +run_app_build "..\scripts\app_build_4862930.vdf" +quit


7. Поставь билд live на ветку. App Admin → SteamPipe → Builds — там появится твой залитый билд. Выставь его live на ветку default (заливка сама live не становится, это вручную). Можно сначала на тестовую ветку, потом на default.
После этого «At Least One Build Configured» позеленеет, а когда добавишь launch option (Installation → Executable = Wrecksmith.exe, OS Windows) — позеленеет и «Launch Options Defined».
Проверка: с твоего аккаунта-разработчика игру можно установить через клиент Steam даже при App Release State = Unavailable — так протестируешь, что билд скачивается и запускается.
Где обычно спотыкаются: неправильный DepotID, в content\ попадает лишнее (исходники/папки bin вместо чистого publish), или забывают выставить билд live.
